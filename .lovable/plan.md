## Obiettivo

Lo snippet incollato è la versione corrente di `SiteMarker.tsx` (stesso markup, JSX perso nel paste) ma usa colori hex hardcoded (`#c0a062`, `#004d61`, `#002838`, `#ffffff`, `rgba(...)`, ecc.). Va reso compliant con il design system del progetto (token HSL semantici in `index.css` + `tailwind.config.ts`), mantenendo identico comportamento, layout, geometria e logica click.

## Cosa cambia

### 1. `src/index.css` — aggiungere token FGB lens

Nuove CSS variables HSL (light + dark):

- `--fgb-gold: 41 39% 57%;`        (≈ #c0a062)
- `--fgb-teal: 192 88% 19%;`       (≈ #004d61)
- `--fgb-teal-ring: 192 86% 28%;`  (≈ #0a6e85)
- `--fgb-navy: 198 100% 11%;`      (≈ #002838)
- `--fgb-navy-ring: 198 64% 28%;`  (≈ #1a5a73)
- `--lens-card: 0 0% 100%;`        (card bianca centrale)
- `--lens-ring-track: 200 19% 95%;` (#eef2f4)
- `--lens-label: 215 16% 47%;`     (#64748b)
- `--lens-unit: 215 20% 65%;`      (#94a3b8)
- `--lens-shadow: 195 100% 11%;`   (per ombra HSL alpha)


### 3. `src/components/dashboard/SiteMarker.tsx` — rewrite compliant

- Sostituire `METRIC_META` per referenziare i token via helper:
  ```ts
  const css = (v: string) => `hsl(var(${v}))`;
  const cssA = (v: string, a: number) => `hsl(var(${v}) / ${a})`;
  ```
  così:
  - energy → `--fgb-gold`
  - air → accent `--fgb-teal`, ring `--fgb-teal-ring`
  - water → accent `--fgb-navy`, ring `--fgb-navy-ring`
- Tutti i `#xxxxxx` / `rgba(...)` inline diventano `css()` / `cssA()` sui token.
- Card centrale: `bg-lens-card`, testo valore `text-fgb-navy`, label `text-lens-label`, unit `text-lens-unit`, track ring `stroke-lens-ring-track`.
- Ombra card e cono: usano `cssA('--lens-shadow', 0.45)` invece di `rgba(0,40,56,0.45)`.
- Geometria, animazioni Framer Motion, callback `onMarkerClick` / `onSphereClick`, fan a 120°, focal point allineato al marker: **invariati**.
- Marker pin (`<img markerPinIcon>`) e gestione hover/timeout: invariati.
